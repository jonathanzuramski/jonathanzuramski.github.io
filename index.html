NFL: Which Plays Win Games?

Author: Jonathan Zuramski

Introduction¶

The National Football League is the United States premiere football league. The NFL is a highly competitive league containing 32 teams, who are looking to optimize their teams strategy to win as many games as possible and move onto the playoffs. NFL games outcomes are highly dependent on the play-by-play performance of teams during a game. Today we are going to investigate which kind of plays generally yield the best performance. Will going for a smaller amount of large play with lots of yardage, or going for short consistent plays end up being more advantageous? We will be looking at the play-by-play data from the 2019 and 2018 NFL seasons, which can be found here in CSV form.

In this tutorial, our goal is to go through the step by step process of data curation, parsing, and management. We are going to look at analysis of our data through machine learning to provide insights in the data to figure out an overall outcome. We wil be using Python 3 as well as slew of libraries

Prerequisite Knowledge¶

Before we start, we will need to know what the two play types we are looking at and how a play functions in football. A play starts off with the Quarterback (the position that throws the ball) starting the play. Before the play is started the play type is decided by a coach or the Quarterback based on what is happening previously. A pass play involves the Quarterback throwing to a receiver downfield. A rush or run is when the Quarterback hands the ball off to a running back who then runs down towards the endzone. There I will be using rush and run interchangeable. In football there are downs; which is the number of attempts to get the ball in the other teams endzone. You only get 4 attempts, unless you move forward ten yards, then the downs are reset. Yards and yardage refers the length down the field, a football field is 100 yards from the bottom of the endzone of one side to the bottom of the other endzone.

The question we are looking at is what play type is better? Getting shorter plays that are more consistent could be better as you have more of a chance of getting the 1st down after reaching 10 yards. But longer plays could give you more chances to get the first down in one shot. We will see.

Let's do a quick overview of groupby, one of the most useful functions to use with a data frame. When grouping like the example above we take values from two columns and tie them together so we can than call a function that works on both. For instance, when we call sum it groups all values in the play_type column, which is now only pass and run. The second argument is the column to be grouped by the different data in the play_type.

So, when we call

'data2019.groupby(['play_type'])['yards_gained'].mean()'

It takes all rows with the play type pass and takes the mean of all pass rows yards_gained. It then does the same for any other value within play_type, our only other one being run.

Making DataFrame for Visuals¶

Lets go ahead and make a dataframe that consists of each games stats organized neatly so we can use it later for graphing. We want to make sure not to collapse to much data when grouping things together. We are going to keep each game seperate, with a row representing a teams stats for the specific game

Keep in mind that when using this data frame later we can reorganize this into another data frame if we are interested in learning from the data in a different format.

#create the new dataframe with the columns we want.
team_data = pd.DataFrame(columns=['game_id','teamID', 'num_runs', 'run_yds', 'num_passes', 'pass_yds', 'win'])

#lets group the yards gained, so that they correspond to an offensive team, a specific game, and a playtype
l1 = data2019.groupby(['game_id', 'posteam', 'play_type'])['yards_gained'].sum().reset_index()

#renaming the column to something easier to understand
l1.rename(columns={'posteam': 'teamId'}, inplace=True)

#creating a grouping with value_counts that gives up the number of plays, so x amount of runs and x amount of passes
# also grouped by gameid and team
l2 = data2019.groupby(['game_id', 'posteam'])['play_type'].value_counts().reset_index(name='number of plays')


# we then drop these into seperate data frames to merge with the master later. 
# passes is a dataframe containing the number of plays for passes
# rushes is a dataframe that contains the number of rushes. 
passes = l2[l2['play_type'] == 'pass']['number of plays'].reset_index().drop(['index'],axis=1)
rushes = l2[l2['play_type'] == 'run']['number of plays'].reset_index().drop(['index'],axis=1)

# drops all the running data and resets the index so it will fit into the new frame properly
# we want passes and rushes to be seperate columns now instead of the same so we will 
# need the new master data frame to be half the size
l3 = l1.iloc[::2,:].reset_index().drop(['index'],axis=1)

# insert all data to each column from the groupbys and organized data frames.
team_data['teamID'] = l3['teamId']
team_data['game_id'] = l3['game_id']
team_data['pass_yds'] = ((l1[(l1['play_type']=='pass')])['yards_gained']).reset_index().drop(['index'],axis=1)
team_data['run_yds'] = ((l1[(l1['play_type']=='run')])['yards_gained']).reset_index().drop(['index'],axis=1)
team_data['num_runs'] = rushes
team_data['num_passes'] = passes
team_data['home_team'] = False
team_data['win'] = False

Machine Learning and Modeling

We have observed the data from just plotting it visually now lets use some libraries to create our own models and see how adding different variable affect the outcome of the prediction. We will use scikit to create the modules below. We are going to make a model for predicting each value. We will do a model for passing yards, then passing yards and number of pass, then we will do the same modules for rushes. From their we will then combine them all and see how accurate we can make our model!

First lets go ahead and look at run yards as that looks to be the most promising single value predictor! We need to hot encode the data for team so they can be processed by the model. Hot encoding turns strings into a numeric value that can go through the functions.

First lets do a prediction with just run_yds and overlay it over the real data. This will give us a linear model with a coefficient and a intercept since its only a single value which will yield a linear model

Linear Model using only Rushing Yards¶

From this module using only run_yds we get a good linear model with an R^2 value of .903 which means the model is reliable!

This is a good model if the data is in fact linear. Looking at the summary provided by the model gives alot of good information on the model itself and how it relates to our data. Looking at the coeff of 0.0044 yards per win shows we need alot of rushing yards!!!

Conclusion

In my analysis of rushing plays vs passing plays I have learned that there are many more factors than need to be taken into account, yet with such simple data points we can somewhat accurately depict how many times a team will win given there isnt a large change in trends in upcoming seasons. I learned a lot about the NLF types of plays and actual performance of teams and how they relate to play picks.

Throughout the entire project there was much evidence that running the ball was a much better option than passing, given the data and visuals. Looking through the rest of the project I believe that more characteristics of a play could be incorporated into the models for a more accurate prediction. For now, I wanted to do a deep dive in the basics and show how much you can learn from the basics play types. Looking through the data for each play-by-play with 256 columns Ive realized in the future I could incorporate more data to make a more accurate model, basing plays off field positioning, what down it is, etc. Overall I really enjoyed this project and will continue using this knowledge to create models to solve problems I face in the real world.
